#### Features

- Four alert types: `success`, `warning`, `danger`, `info`
- Optional `title`, `description`, and `icon`
- Action area for custom buttons or links
- Closable via a built-in close button
- Fully customizable via `className` and `style`


#### Basic info Alert

Use an info alert when you want to highlight a neutral, helpful message that doesn’t require user action. It works best for general notifications.

<Alert
  title="Heads up!"
  description="This is an informational message."
  type="info"
/>

```tsx
<Alert
  title="Heads up!"
  description="This is an informational message."
  type="info"
/>
```


#### Success Alert with Icon

Success alerts confirm positive outcomes, such as saving changes or completing an action. Adding an icon helps users quickly recognize the status.

<Alert
  title="Success!"
  description="Your changes have been saved."
  type="success"
  icon={<IconRosetteDiscountCheck />}
/>

```tsx
import { IconRosetteDiscountCheck } from "@tabler/icons-react";

<Alert
  title="Success!"
  description="Your changes have been saved."
  type="success"
  icon={<IconRosetteDiscountCheck />}
/>
```

#### Warning Alert with Action

Warnings notify users about potential problems or limitations.
Adding an `action` (e.g., “Upgrade Plan”) helps guide the user toward solving the issue.

<Alert
  title="Storage almost full"
  description="You are using 95% of your storage capacity."
  type="warning"
  isClosable
  action={<Button size="s">Upgrade Plan</Button>}
/>

```tsx
<Alert
  title="Storage almost full"
  description="You are using 95% of your storage capacity."
  type="warning"
  isClosable
  action={<Button size="s">Upgrade Plan</Button>}
/>
```


#### Closable Alert

Use closable alerts when the message is temporary or dismissible. This gives users control and reduces interface clutter once they have acknowledged the message.

<Alert
  title="Session expired"
  description="Please log in again to continue."
  type="danger"
  isClosable
  onClose={() => alert("Alert closed")}
/>

```tsx
<Alert
  title="Session expired"
  description="Please log in again to continue."
  type="danger"
  isClosable
  onClose={() => console.log("Alert closed")}
/>
```


#### Sized variants

The `size` prop adjusts the density of the alert.

- `s` works best for compact layouts or inline usage
- `m` is the default and most commonly used
- `l` is more prominent and ideal for critical or system-wide notifications

<Flex gap={12} direction="column">
  <Alert size="s" title="Compact" description="Small alert" />
  <Alert size="m" title="Default" description="Medium alert" />
  <Alert size="l" title="Large" description="Prominent alert" />
</Flex>

```tsx

  <Alert size="m" title="Default" description="Medium alert" />
  <Alert size="l" title="Large" description="Prominent alert" />
</Flex>
```


#### Custom styling

  title="Default" description="Medium alert"

#### Best Practices

  - Prioritize clarity: Always provide a clear `title` for important messages.
  - Keep it concise: Limit `description` text to 1–2 short sentences.
  - Use severity wisely: Reserve `type="danger"` for errors or blocking issues; use warning for cautionary notices.
  - Don’t overload actions: Add an `action` button only when immediate user response is required.
  - Be consistent with icons: Use icons that clearly represent the message type and keep them at a consistent size (16–20px recommended).

#### Troubleshooting

  - Alert not closing: Make sure `isClosable` is set and `onClose` is handled properly.
  - Missing styles: Verify your app is wrapped with `QuenUIProvider`.
  - Icon alignment issues: Use icons with square dimensions to maintain vertical alignment.
