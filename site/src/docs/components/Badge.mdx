import { Badge, Flex, Avatar } from "@quen-ui/components";
import { IconBell, IconLock, IconLockOpen2 } from "@tabler/icons-react";

#### Features

- Multiple size variants (`xs`, `s`, `m`, `l`)
- Predefined semantic colors (`success`, `warning`, `danger`, `secondary`)
- Custom color support via `color` prop
- Optional left and right icons (`leftContent`, `rightContent`)
- Works with text, numbers, or custom content
- Accessible and responsive by default

#### Basic usage

Use a badge to display a simple count or highlight. This is the most common form, often used for notification counters or tags.

<Badge text="3">
  <Avatar name="Alan Desmont" />
</Badge>

```tsx
<Badge text="3">
  <Avatar name="Alan Desmont" />
</Badge>
```

#### Status badges

Status badges communicate the current state of an item. Use semantic colors (`success`, `warning`, `danger`, `secondary`) to ensure consistent meaning across the application.

<Flex gap={8}>
  <Badge color="secondary" text="Draft" />
  <Badge color="success" text="Active" />
  <Badge color="warning" text="Pending" />
  <Badge color="danger" text="Expired" />
  <Badge color="disabled" text="Disabled" />
</Flex>

```tsx
<Flex gap={8}>
  <Badge color="secondary" text="Draft" />
  <Badge color="success" text="Active" />
  <Badge color="warning" text="Pending" />
  <Badge color="danger" text="Expired" />
  <Badge color="disabled" text="Disabled" />
</Flex>
```

#### With icons

You can enhance clarity by pairing badges with icons. This works well for categories, alerts, or interactive UI elements.

<Badge
  leftContent={<IconBell size={14} />}
  color="danger"
  className="p-remove-margin"
  text="Alerts"
/>

```tsx
<Badge
  leftContent={<IconBell size={14} />}
  color="danger"
  className="p-remove-margin"
  text="Alerts"
/>
```

#### Notification counter

A badge can also be used as a compact numeric indicator, usually placed next to icons or navigation items.

<Badge color="danger" text={5} />

```tsx
<Badge color="danger" text={5} />
```

#### Custom colors

In addition to semantic variants, you can apply custom colors. This is useful for branding or special tags.

<Badge color="#6C5CE7" text="Premium" />

```tsx
<Badge color="#6C5CE7" text="Premium" />
```

#### Size variants

Badges come in four sizes:

- `xs` → ultra-compact, for dense UI (tables, lists)
- `s` → small, for labels or secondary info
- `m` → medium (default), for general use
- `l` → large, for prominent tags or labels

<Flex gap={8} align="center">
  <Badge size="xs" text="xs" />
  <Badge size="s" text="s" />
  <Badge size="m" text="m" />
  <Badge size="l" text="l" />
</Flex>

```tsx
<Flex gap={8} align="center">
  <Badge size="xs" text="xs" />
  <Badge size="s" text="s" />
  <Badge size="m" text="m" />
  <Badge size="l" text="l" />
</Flex>
```

#### Complex content

Badges are flexible and can contain icons on both sides of the text, making them suitable for rich labels or access control indicators.

<Badge
  leftContent={<IconLock size={12} />}
  rightContent={<IconLockOpen2 size={12} />}
  text="Private"
  className="p-remove-margin"
/>

```tsx
<Badge
  leftContent={<IconLock size={12} />}
  rightContent={<IconLockOpen2 size={12} />}
  text="Private"
/>
```

#### Best Practices

- Use for short labels (1-3 words)
- Reserve red (`danger`) for urgent statuses
- Pair icons with text for clarity
- Maintain consistent badge usage across application
