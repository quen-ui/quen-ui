#### Features

- Five visual variants (`info`, `success`, `warning`, `danger`, `neutral`)
- Optional title and rich description content
- Action slot for buttons or links
- Auto icons per variant or custom icon support
- Responsive: `fullWidth` mode for fluid layouts
- Polymorphic: render as any element via `as` prop

#### Basic Usage

A standard banner with description text.

<Banner description="This is a standard informational banner with default settings." />

```tsx
<Banner description="This is a standard informational banner with default settings." />
```

#### With Title

/>

  <Banner variant="warning" description="Please review before proceeding." />
  <Banner variant="danger" description="Critical error occurred." />
  <Banner variant="neutral" description="Neutral context message." />

#### With Action

Include interactive elements in the `action` prop.

<Banner
  variant="info"
  description="New features are available. Learn what's new."
  action={
    <Button size="s" view="link">
      Learn more
    </Button>
  }
/>

#### Dismissible

Enable `dismissible` to allow users to close the banner.

<Banner
  variant="warning"
  title="Browser Update Required"
  description="For the best experience, please update your browser."
  dismissible
  onClose={() => console.log("Banner dismissed")}
/>

```tsx
<Banner
  variant="warning"
  title="Browser Update Required"
  description="For the best experience, please update your browser."
  dismissible
  onClose={() => console.log("Banner dismissed")}
/>
```

#### Best Practices

 - Use variant semantically: danger only for errors, success for confirmations
 - Keep titles concise; use description for details
 - Limit to one action per banner to avoid decision fatigue
 - Make dismissible banners non-critical; essential alerts should persist

#### Troubleshooting
 - Banner not visible: Check if onClose was called or state controls visibility
 - Icon not showing: Ensure icon prop or variant has associated icon
 - Action misaligned: Wrap action content in appropriate container
 - SSR hydration: Ensure consistent rendering between server/client
