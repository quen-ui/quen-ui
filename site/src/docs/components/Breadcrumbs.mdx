#### Features

- Displays navigation path with clickable items
- Configurable item properties: `label`, `href`, `icon`, `onClick`
- Supports custom data structures with mapping functions
- Adjustable separator (default: `/`)
- Multiple size variants (`xs`, `s`, `m`, `l`)
- Optional icon-only root element
- Fully accessible with ARIA support and keyboard navigation

#### Basic Usage

Breadcrumbs are collected from page names and links to them. You can add an icon from the icon library to any page.

All this can be described in the items array of the `IBreadcrumbItemDefault` type, it consists of:

- label — page title,
- icon — icon from [icon library],
- href — page link,
- onClick — click event,
- className - additional classname

```ts
export interface IBreadcrumbItemDefault {
  label: string;
  icon?: React.ReactNode;
  href?: string;
  onClick?: React.MouseEventHandler<HTMLAnchorElement>;
  className?: string;
}

export type TBreadcrumbPropGetItemLabel<ITEM> = (item: ITEM) => string;
export type TBreadcrumbPropGetItemHref<ITEM> = (
  item: ITEM
) => string | undefined;
export type TBreadcrumbPropGetItemIcon<ITEM> = (item: ITEM) => React.ReactNode;
export type TBreadcrumbPropGetItemOnClick<ITEM> = (
  item: ITEM
) => React.MouseEventHandler | undefined;
export type TBreadcrumbsPropOnItemClick<ITEM = IBreadcrumbItemDefault> = (
  item: ITEM,
  e: React.MouseEvent
) => void;
export type TBreadcrumbsPropGetItemClassname<ITEM> = (
  item: ITEM
) => string | undefined;
```

<BaseUsage />

```tsx
function PageNavigation() {
  const items = [
    { id: 1, label: "Home", href: "/" },
    { id: 2, label: "Products", href: "/products" },
    { id: 3, label: "Current Page" }
  ];

  return <Breadcrumbs items={items} />;
}
```

#### Custom items

Breadcrumbs can adapt to custom data structures by using `getItemLabel`, `getItemHref`, and other mapping props. This is helpful when integrating with APIs or existing data models.

<CustomItems />

```tsx
const items = [
  { id: 1, name: 'Home', path: '/' },
  { id: 2, name: 'Blog', path: '/blog' },
  { id: 3, name: 'Post' }
];

<Breadcrumbs
  items={items}
  getItemLabel={(item) => item.name}
  getItemHref={(item) => item.path}
/>
```

#### With custom separator

By default, breadcrumbs use / as a separator. You can replace it with any string or component (e.g., →, >).

<CustomSeparator />

```tsx
const items = [
  { id: 1, name: 'Home', path: '/' },
  { id: 2, name: 'Blog', path: '/blog' },
  { id: 3, name: 'Post' }
];

<Breadcrumbs
  items={items}
  getItemLabel={(item) => item.name}
  separator={"→"}
/>
```

#### With icons

Items can include icons for better recognition. You can also set `isOnlyIconRoot` to show only an icon for the root element (commonly "Home").

<WithIcons />

```tsx
 const items = [
    { id: 1, label: 'Home', icon: <IconHome /> },
    { id: 2, label: 'Settings', icon: <IconSettings /> }
  ];

  return (
    <Breadcrumbs
      items={items}
      onlyIconRoot
    />
  );
```

#### Size variants

Breadcrumbs support multiple sizes (`xs`, `s`, `m`, `l`). Smaller sizes fit compact UIs like toolbars, while larger ones are suitable for spacious layouts.

<Sizes />

```tsx
  const items = [
    { id: 1, label: "Home", icon: <IconHome width={16} /> },
    { id: 2, label: "Settings", icon: <IconSettings width={16} /> }
  ];

 <Flex gap={8} direction="column">
  <Breadcrumbs size="xs" items={items} onlyIconRoot />
  <Breadcrumbs size="s" items={items} onlyIconRoot />
  <Breadcrumbs size="m" items={items} onlyIconRoot />
  <Breadcrumbs size="l" items={items} onlyIconRoot />
 </Flex>
```

#### Best Practices

  - Keep trails short: Limit breadcrumbs to 2–5 items for readability.
  - Make only the last item inactive: All previous items should be clickable.
  - Use clear labels: Avoid abbreviations unless they are universally understood.
  - Mobile responsiveness: Consider truncating long breadcrumb trails on smaller screens.
  - Use icons sparingly: Icons should add clarity, not noise.

#### Troubleshooting

  - Links not working: Ensure getItemHref returns a valid URL or href is set.
  - Custom data not rendering: Double-check getItemLabel and mapping functions.
  - Separator not showing: Pass a valid string or React node to separator.
  - Accessibility warnings: Make sure label is always provided, even when using icons.
