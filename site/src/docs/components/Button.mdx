#### Features
  - Multiple visual styles (`primary`, `secondary`, `success`, `warning`, `danger`, `link`, `ghost`, `icon`)
  - Three size variants (`s`, `m`, `l`)
  - Supports icons on the left or right side
  - Loading state with customizable loader
  - Disabled state for non-interactive buttons
  - `fullWidth` option for responsive layouts
  - Can render as different elements via `as` prop (`button`, `a`, `div`, custom component)
  - Accessible: keyboard navigation, ARIA attributes, focus states
  - **ButtonGroup** for grouping related actions with collapsed borders and shared styling

#### Basic Usage

A standard button with medium size (`size="m"`) and primary style (`view="primary"`).

<Button onClick={() => console.log("Clicked!")}>
  <span>Click Me</span>
</Button>

#### Loading state

The `loading` prop shows a spinner and prevents user interaction.

<Button loading>
  <span>Processing...</span>
</Button>

```tsx
function SubmitButton() {
  const [isLoading, setLoading] = useState(false);

  const handleClick = () => {
    setLoading(true);
    // Simulate API call
    setTimeout(() => setLoading(false), 2000);
  };

  return (
    <Button loading={isLoading} onClick={handleClick}>
      {isLoading ? "Processing..." : "Submit"}
    </Button>
  );
}
```

#### Button.Group

`Button.Group` is a container component that visually combines related `Button` elements into a single cohesive control. It automatically collapses borders between buttons, manages border-radius (rounded only on the outer edges), and ensures proper hover/focus states.

The component supports **prop inheritance**: you can set `size` or `view` on the group, and all child buttons will adopt those values — unless a button explicitly overrides them with its own props.

##### Basic group

Group related actions together, such as "Copy", "Share", and "Download".

<Button.Group aria-label="File actions">
  <Button leftContent={<IconCopy size={16} />}>Copy</Button>
  <Button leftContent={<IconShare size={16} />}>Share</Button>
  <Button leftContent={<IconDownload size={16} />}>Download</Button>
</Button.Group>

```tsx
<Button.Group aria-label="File actions">
  <Button leftContent={<IconCopy size={16} />}>Copy</Button>
  <Button leftContent={<IconShare size={16} />}>Share</Button>
  <Button leftContent={<IconDownload size={16} />}>Download</Button>
</Button.Group>
```

<Button.Group view="secondary" size="s" aria-label="View options">
  <Button>Day</Button>
  <Button>Week</Button>
  <Button>Month</Button>

##### Overriding individual buttons

A child button can override the group's `view` or `size` by specifying its own props. This is useful for highlighting a primary action within a group.

<Button.Group view="secondary" size="m" aria-label="Mixed actions">
  <Button>Cancel</Button>
  <Button>Save as draft</Button>
  <Button view="primary">Publish</Button>
</Button.Group>

```tsx

```

#### Best practices

  - Use `primary` only for the main action on a page
  - Use `secondary/ghost` for less prominent actions
  - Prefer `danger` only for destructive actions
  - Add icons only when they improve clarity
  - Use `fullWidth` in forms or narrow layouts
  - Use `Button.Group` for **related joint actions** (e.g., "Copy" + "Share"). For **mutually exclusive choices** (e.g., "Day / Week / Month" toggle), prefer a dedicated `SegmentedControl` component instead

#### Troubleshooting

  - Click handler not firing: Ensure button is not `disabled` or `loading`
  - Loader not visible: Check `loaderProps.size` and color contrast
  - Accessibility issues: Avoid using only `icon` view without `aria-label`
  - Wrong element rendered: Confirm `as` matches valid HTML element or component
  - Double borders in a group: Ensure you're using `Button.Group` rather than a plain `Flex` — the group handles border collapsing automatically
  - Hover border cut off by neighbor: This is handled internally via `z-index`; if it still occurs, check that no parent element sets `overflow: hidden`
