#### Basic usage

A standard card with a header (title) and main content area.

<Card title="Basic Card">
  <Text>This is a simple card component with title and content</Text>
</Card>

```tsx
import { Card, Text } from "@quen-ui/components";

function BasicCard() {
  return (
    <Card title="Basic Card">
      <Text>This is a simple card component with title and content</Text>
    </Card>
  );
}
```

#### Card without title

The `title` prop is optional. You can render a simple container-like card with only body content.

<Card>
  <Text>This card has no title, just content</Text>
</Card>

```tsx
<Card>
  <Text>This card has no title, just content</Text>
</Card>
```

#### Card with Cover Image

Cards can display a `cover` element (usually an image) at the top.

<Card
  title="Vacation Photo"
  cover={<img src={Avatar2} alt="Beach vacation" width={350} />}
>
  <Text>Our team retreat in Hawaii</Text>
</Card>

```tsx
<Card
  title="Vacation Photo"
  cover={<img src="/vacation.jpg" alt="Beach vacation" style={350} />}>
  <Text>Our team retreat in Hawaii</Text>
</Card>
```

#### Card with actions

Use the `actionContent` prop for footer actions (typically Save/Cancel buttons).

<Card
  title="Settings"
  actionContent={
    <div style={{ display: "flex", gap: 8 }}>
      <Button size="s">Save</Button>
      <Button size="s" view="ghost">
        Cancel
      </Button>
    </div>
  }>
  <Text>Configure your preferences</Text>
</Card>

```tsx
<Card
  title="Settings"
  actionContent={
    <div style={{ display: "flex", gap: 8 }}>
      <Button size="s">Save</Button>
      <Button size="s" view="ghost">
        Cancel
      </Button>
    </div>
  }>
  <Text>Configure your preferences</Text>
</Card>
```

#### Card with Header Extra

Use `extra` for additional controls in the header (e.g., links or secondary actions).

<Card
  title="Notifications"
  extra={
    <Button view="link" size="s">
      Mark all read
    </Button>
  }>
  <Text>You have 3 new notifications</Text>
</Card>

```tsx
<Card
  title="Notifications"
  extra={
    <Button view="link" size="s">
      Mark all read
    </Button>
  }>
  <Text>You have 3 new notifications</Text>
</Card>
```

#### Card with Left Content in Header

The `leftContent` prop allows placing an avatar, icon, or any element to the left of the title.

<Card
  title="Profile"
  leftContent={<img src={Avatar2} alt="Avatar" width={32} height={32} />}
>
  <Text>John Doe – Software Engineer</Text>
</Card>

```tsx
<Card
  title="Profile"
  leftContent={
    <img
      src="/vacation.jpg"
      alt="Beach vacation"
      alt="Avatar"
      width={32}
      height={32}
    />
  }
>
  <Text>John Doe – Software Engineer</Text>
</Card>
```

#### Size variants

#### Accessibility

- Semantic HTML structure
- Proper heading hierarchy
- Keyboard-navigable interactive elements
- ARIA attributes for actionable items

#### Best Practices

###### 1. Content Organization:

- Use for discrete chunks of content
- Group related information together
- Avoid nesting cards more than 2 levels deep

###### 2. Actions Placement:

- Primary actions in footer (`actionContent`)
- Secondary actions in header (`extra`)
- Important actions should be buttons

###### 3. Performance:

- Optimize cover images
- Avoid complex computations in render
- Consider lazy-loading offscreen cards
