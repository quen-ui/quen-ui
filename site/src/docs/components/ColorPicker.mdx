import { ColorPicker } from "@quen-ui/components";

#### Basic usage

A simple color picker with default format (hex) and full UI: spectrum, sliders, inputs, and presets.

<ColorPicker />

```tsx
import { ColorPicker } from "@quen-ui/components";

<ColorPicker />;
```

#### Controlled value

The component behaves as a controlled input, fully driven by external React state.

<ColorPicker />

```tsx
const [color, setColor] = useState("#ff00aa");

<ColorPicker value={color} onChange={setColor} />;
```

#### With change completion callback

`onChangeComplete` only fires when the user finishes an interaction — useful for debouncing updates to backend/UI.

<ColorPicker
  defaultValue="#ff0000"
  onChange={(c) => console.log("Live:", c)}
  onChangeComplete={(c) => console.log("Done:", c)}
/>

```tsx
<ColorPicker
  defaultValue="#ff0000"
  onChange={(c) => console.log("Live:", c)}
  onChangeComplete={(c) => console.log("Done:", c)}
/>
```

#### Using preset colors

Shows predefined swatches that users can directly click to apply a color.

<ColorPicker
  presets={["#ff0000", "#00ff00", "#0000ff", "rgb(0,0,0)"]}
/>

```tsx
<ColorPicker
  presets={["#ff0000", "#00ff00", "#0000ff", "rgb(0,0,0)"]}
/>
```

#### Hiding inputs (palette-only mode)

Removes input fields, leaving only the visual color selection tools — ideal for compact or simplified UIs.

<ColorPicker hideInputs format="rgba" />

```tsx
<ColorPicker hideInputs format="rgba" />
```

#### Best practices

  - Prefer `ColorPicker.Input` for Forms. It integrates better with layout and users are accustomed to input-triggered popovers.
  - Use `onChangeComplete` to avoid performance issues. Real-time color changes (`onChange`) can be expensive when updating large previews or canvases.
  - Keep format consistent across app. Avoid mixing hex/rgb/hsl formats arbitrarily; pick a standard format per project.
  - Provide presets for frequently used colors.
  - Hide inputs in simple UIs. If users don’t need manual color typing, `hideInputs` keeps the UI cleaner.
