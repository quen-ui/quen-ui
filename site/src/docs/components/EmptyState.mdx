#### Custom Content

Includes additional interactive content, such as buttons or links, to guide the user.

<EmptyState
  title="No Projects"
  description="Create your first project to get started.">
  <Button onClick={() => alert("Create project")}>Create Project</Button>
</EmptyState>

```tsx
<EmptyState
  title="No Projects"
  description="Create your first project to get started.">
  <Button onClick={() => alert("Create project")}>Create Project</Button>
</EmptyState>
```

#### Best Practices

  - Keep it concise – The title and description should clearly explain why the empty state is shown.
  - Provide guidance – Include actions such as buttons or links to help users move forward.
  - Use visual cues – Images or illustrations help make the empty state more engaging.
  - Consistency – Ensure empty states across the application follow the same layout and style.
