#### Basic usage

<BaseForm />

```tsx
import { Form, Button, TextField, Checkbox } from "@quen-ui/components";

const BaseForm = () => {
  const form = Form.useForm({
    initialValues: { user: { email: "you@example.com" } }
  });

  const handleSubmit = (values: any) => {
    console.log(values);
  };

  return (
    <Form form={form} onFinish={handleSubmit}>
      <Form.Field
        name="user.email"
        rules={[{ required: true }]}
        validateTrigger="onChange"
        validate={(val) =>
          val.includes("test") ? ["Email must not contain 'test'"] : undefined
        }>
        <TextField label="Email" placeholder="you@example.com" />
      </Form.Field>

      <Form.Field name="password" rules={[{ required: true }]}>
        <TextField label="Password" />
      </Form.Field>

      <Form.Field name="remember" valuePropName="checked">
        <Checkbox label="Remember me" />
      </Form.Field>

      <Button type="submit">Submit</Button>
    </Form>
  );
};
```

#### Render props pattern

For maximum flexibility, `Field` supports the render props pattern. This is ideal when you need custom layout, conditional rendering, or access to field metadata like `touched` and `dirty`.

<RenderPropsForm />

```tsx
const RenderPropsForm = () => {
  const form = Form.useForm();

  return (
    <Form form={form}>
      <Form.Field
        name="email"
        rules={[{ required: true, type: "email" }]}
        validateTrigger="onBlur">
        {({ value, onChange, onBlur, error, touched, dirty, required }) => (
          <Flex direction="column">
            <TextField
              value={value}
              onChange={onChange}
              onBlur={onBlur}
              label="Email"
              placeholder="you@example.com"
              required={required}
            />
            {touched && error && <span className="error-message">{error}</span>}
            {dirty && <span className="badge">Modified</span>}
          </Flex>
        )}
      </Form.Field>
    </Form>
  );
};
```

#### Touched and Dirty states

Track user interaction (`touched`) and value changes (`dirty`) to build better UX. Errors are typically shown only after a field is touched, and submit buttons are enabled only when the form is dirty.

<TouchedDirtyForm />

```tsx
const TouchedDirtyForm = () => {
  const form = Form.useForm({
    initialValues: { name: "", email: "" }
  });

  const hasChanges = form.isFieldsDirty();
  const touchedFields = form.getTouchedFields();

  return (
    <Form form={form} onFinish={(values) => console.log(values)}>
      <Form.Field name="name" rules={[{ required: true }]}>
        {({ value, onChange, onBlur, error, touched }) => (
          <div>
            <TextField
              value={value}
              onChange={onChange}
              onBlur={onBlur}
              label="Name"
              error={touched ? error: undefined}
            />
          </div>
        )}
      </Form.Field>

      <Form.Field name="email" rules={[{ required: true, type: "email" }]}>
        <TextField label="Email" />
      </Form.Field>

      <div className="actions">
        <Button type="button" onClick={() => form.resetFields(["email"])}>
          Reset Email Only
        </Button>
        <Button
          type="submit"
          disabled={!hasChanges}
          loading={form.isSubmitting}>
          Save Changes
        </Button>
      </div>

      <p>Touched fields: {touchedFields.join(", ") || "none"}</p>
    </Form>
  );
};
```

##### Available methods:

```tsx
// Touched
form.isFieldTouched("email");        // boolean
form.isFieldsTouched();              // any field touched?
form.isFieldsTouched(["a", "b"]);    // any of these touched?
form.getTouchedFields();             // string[]
form.touchField("email");            // mark as touched
form.touchFields(["a", "b"]);        // mark multiple

// Dirty
form.isFieldDirty("email");          // differs from initialValues?
form.isFieldsDirty();                // any field dirty?
form.getDirtyFields();               // string[]
```

#### Watching field values

Use `Form.useWatch` to observe changes to specific fields.

<WatchForm />

```tsx
import { Form, TextField } from "@quen-ui/components";

const WatchForm = () => {
  const form = Form.useForm();
  const email = Form.useWatch("email", form);

  return (
    <Form form={form}>
      <Form.Field name="email">
        <TextField placeholder="Enter email" label="Email" />
      </Form.Field>
      <p>Live preview: {email}</p>
    </Form>
  );
};
```

##### Validation triggers

Control when validation runs using `validateTrigger`. This can be set globally on `<Form>` or per-field on `<Field>`.

```tsx
// Validate on every keystroke (default)
<Form.Field name="email" validateTrigger="onChange">

// Validate only on blur (better UX for expensive validators)
<Form.Field name="email" validateTrigger="onBlur">

// Validate only on form submit
<Form.Field name="comment" validateTrigger="onSubmit">

// Multiple triggers
<Form.Field name="password" validateTrigger={["onChange", "onBlur"]}>

// Global default for all fields
<Form form={form} validateTrigger="onBlur">
```

#### Async validation with debounce

When validation requires an API call (e.g., checking email uniqueness), use `validateDebounce` to avoid spamming the server on every keystroke.

```tsx
const DebounceValidationForm = () => {
  const form = Form.useForm();

  const checkUsername = async (value: string) => {
    if (!value) return undefined;
    const isTaken = await api.checkUsername(value);
    return isTaken ? ["Username is already taken"]: undefined;
  };

  return (
    <Form form={form}>
      <Form.Field
        name="username"
        validate={checkUsername}
        validateTrigger="onChange"
        validateDebounce={500}
      >
        {({ value, onChange, error, touched }) => (
          <div>
            <TextField
              value={value}
              onChange={onChange}
              label="Username"
              placeholder="Choose a username"
            />
            {form.isValidating && <span>Checking...</span>}
            {touched && error && <span className="error">{error}</span>}
          </div>
        )}
      </Form.Field>
    </Form>
  );
};
```

#### Dynamic field arrays

Manage arrays of fields with `Form.FieldArray`.

<DynamicFieldsForm />

```tsx
import { Form, TextField } from "@quen-ui/components";

const DynamicFieldsForm = () => {
  const form = Form.useForm();
  return (
    <Form form={form}>
      <Form.FieldArray name="users">
        {({ add }, fields) => (
          <>
            {fields.map((field, index) => (
              <Form.Field key={index} name="name">
                <TextField
                  placeholder="Enter user name"
                  label={`User ${index + 1}`}
                />
              </Form.Field>
            ))}

            <Button type="button" onClick={() => add()}>
              Add User
            </Button>
          </>
        )}
      </Form.FieldArray>
    </Form>
  );
};
```

const CustomValidationMessagesForm = () => {

#### Features

- Type-safe form handling with TypeScript generics
- Nested and array field support (user.name, friends[0].name)
- Render Props pattern for maximum flexibility
- Touched/Dirty tracking for better UX
- Custom validation (no schema library required)
- Async validation with debounce for API checks
- Reactive subscriptions via useWatch
- Loading states (isSubmitting, isValidating)
- Controlled triggers for value and validation updates
- Reset and initial values
- Simple integration with UI components (TextField, Checkbox, Button)

#### When to use

- When you need full control over field state and validation
- When working with nested or dynamic form structures
- When using a lightweight solution without external schema libraries
- When you need fine-grained control over validation timing and UX

#### Best Practices

- Use Form.useForm() for shared state across components
- Wrap your submit button in `<button type="submit">` to trigger submission naturally
- Use FieldArray for any repeated or dynamic input lists
- Combine with UI components like TextField, Select, Switch for consistent design
- Keep validation messages concise and human-readable
- Show errors only after touched to avoid overwhelming users
- Disable submit button when !isFieldsDirty() to prevent unnecessary API calls
- Use validateDebounce for async validators to avoid API spam
- Use validateTrigger="onBlur" for expensive validators (API checks)
- Use triggerValidation for multi-step forms to validate only current step
- Use deep watching with useWatch("parent", form) to observe nested changes
