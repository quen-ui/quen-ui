import { Highlight } from "@quen-ui/components";

###### Use cases include:

- Highlighting search results
- Emphasizing keywords in text snippets
- Displaying OCR or NLP-based fragments
- Highlighting ranges coming from backend indexing systems
- Inline highlighting within typography components

#### Basic usage

<Highlight query="dolor">
  Lorem ipsum dolor sit amet, consectetur adipiscing elit. Quisque sit.
</Highlight>

```tsx
<Highlight query="dolor">
  Lorem ipsum dolor sit amet, consectetur adipiscing elit. Quisque sit.
</Highlight>
```

#### Multiple queries

Here, multiple search terms are highlighted simultaneously. This is useful for search interfaces and keyword previews.

<Highlight query={["Lorem", "dolor", "elit"]}>
  Lorem ipsum dolor sit amet, consectetur adipiscing elit. Quisque sit.
</Highlight>

```tsx
<Highlight query={["Lorem", "dolor", "elit"]}>
  Lorem ipsum dolor sit amet, consectetur adipiscing elit. Quisque sit.
</Highlight>
```

#### Case-Sensitive search

Only the exact case-matched “Lorem” is highlighted. Use this when case matters, such as acronyms or code-like content.

<Highlight query="Lorem" caseSensitive>
  Lorem ipsum dolor sit amet, consectetur adipiscing elit. Quisque sit lorem.
</Highlight>

```tsx
<Highlight query="Lorem" caseSensitive>
  Lorem ipsum dolor sit amet, consectetur adipiscing elit. Quisque sit lorem.
</Highlight>
```

#### Whole word matching

Highlights only standalone words, avoiding matches inside longer words. Ideal for precise semantic highlighting.

<Highlight query="art" splitByWords>
  The architecture article starts with an artistic introduction.
</Highlight>

```tsx
<Highlight query="art" splitByWords>
  The architecture article starts with an artistic introduction.
</Highlight>
```

#### Custom highlight tag

Instead of the default `<mark>`, this example uses `<strong>`. Useful for simple typography-based emphasis.

<Highlight query="Lorem" highlightTag="strong">
  Lorem ipsum dolor sit amet, consectetur adipiscing elit. Quisque sit lorem.
</Highlight>

```tsx
<Highlight query="Lorem" highlightTag="strong">
  Lorem ipsum dolor sit amet, consectetur adipiscing elit. Quisque sit lorem.
</Highlight>
```

#### Highlighting explicit ranges

This example shows how backend-provided or algorithmically calculated ranges override query-based matching.

<Highlight
  ranges={[
    { start: 0, end: 4 },
    { start: 10, end: 16 }
  ]}>
  Lorem ipsum dolor sit amet, consectetur adipiscing elit. Quisque sit lorem.
</Highlight>

```tsx
<Highlight
```

#### Best practice

- Prefer Query Matching for Search-Like Highlighting
- Use `ranges` for Precise / Programmatic Highlighting
- Don’t Overuse Custom Components
- Avoid Passing Very Large Text
- Use maxChunks for User-Supplied Queries
