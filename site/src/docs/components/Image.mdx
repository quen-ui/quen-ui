####  Error handling

```tsx
const SafeImage = () => {
  const [error, setError] = useState(false);

  return (
    <Image
      src={error ? '' : '/unreliable-source.jpg'}
      alt="Important content"
      fallback="/fallback.jpg"
      onError={() => {
        console.error('Image failed to load');
        setError(true);
      }}
    />
  );
}
```

Demonstrates advanced error handling. Logs errors and provides a fallback image when the primary source fails.

#### Best practices

- Product images in e-commerce
- User avatars and profile pictures
- Content-heavy pages (blogs, galleries)
- Applications requiring image previews
- Dynamic image sources from APIs
- Use WebP for modern browsers
- Provide JPEG fallback
- Use PNG for transparency needs
- Compress images (aim for 70-85% quality)
- Use CDN for image delivery
- Implement lazy loading below the fold

#### Accessibility

- Descriptive for content images
- Empty (`alt=""`) for decorative images
- Avoid "image of..." redundancy
- Provide captions for complex images
- Use ARIA attributes when needed
- Ensure sufficient color contrast
