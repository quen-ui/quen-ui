#### Basic number input

A simple quantity selector with minimum and maximum constraints. Users can adjust the value with keyboard arrows or stepper buttons.

<InputNumber label="Quantity" min={1} max={10} defaultValue={1} />

```tsx
import { InputNumber } from "@quen-ui/components";

function QuantitySelector() {
  const [quantity, setQuantity] = useState(1);

  return (
    <InputNumber
      label="Quantity"
      value={quantity}
      onChange={setQuantity}
      min={1}
      max={10}
    />
  );
}
```

#### Currency input with formatting

Handles numeric values as formatted currency strings.
The `formatter` controls how the number is displayed (e.g. `$1,000.00`), while the `parser` ensures that only raw numeric values are stored.

<InputNumber
  label="Price"
  defaultValue={25.99}
  formatter={(value) => `$ ${value}`.replace(/\B(?=(\d{3})+(?!\d))/g, ",")}
  parser={(value) => parseFloat(value?.replace(/\$\s?|(,*)/g, ""))}
  decimalSeparator="."
  step={0.01}
/>

#### Percentage input with constraints

Accepts only values between `0–100`. The `rightContent` adds a percentage symbol, while validation shows an error if the limit is exceeded.

<InoutNumberWithRightContent />

```tsx
<InputNumber
  label="Discount Rate"
  min={0}
  max={105}
  step={5}
  isAllowNegative={false}
  rightContent="%"
  placeholder="0-100"
  error={value > 100 ? "Cannot exceed 100%" : undefined}
/>
```

#### Clearable input with validation

Required input with clear button support. Displays an error when the field is empty and provides a currency icon prefix.

<PriceInput />

```tsx
const PriceInput = () => {
  const [price, setPrice] = useState<number | null>(null);

  return (
    <InputNumber
      label="Product Price"
      value={price}
      onChange={(value) => setPrice(value ? Number(value) : null)}
      isClearable
      isRequired
      error={price === null && "Price is required"}
      leftContent={<IconCurrencyDollar />}
    />
  );
};
```

#### Custom decimal formatting

European-style formatting using a comma `(",")` as decimal separator. The `formatter`/`parser` pair ensures correct conversion between user input and stored value.

<InputNumber
  label="European Format"
  decimalSeparator=","
  formatter={(value) => value?.toString().replace(".", ",") || ""}
  parser={(value) => parseFloat(value?.replace(",", "."))}
  placeholder="0,00"
/>

#### Best practices

  - Currency amounts
  ```ts
    formatter: value => `$${value.toFixed(2)}`
  ```
  - Percentages:
  ```ts
    formatter: value => `${value}%`
  ```
  - Large Numbers:
  ```ts
    formatter: value => value.toLocaleString()
  ```
  - Negative Values:
  ```ts
    formatter: value => value < 0 ? `-$${Math.abs(value)}` : `$${value}`
  ```

Use cases include:
  - Product quantities
  - Prices and currency amounts
  - Discounts and percentages
  - Scientific values (temperature, measurements)
  - Configuration values (settings, limits)
