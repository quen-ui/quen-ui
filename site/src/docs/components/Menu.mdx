#### Dynamic items with custom getters

Adapts the Menu to a custom data model.
  - Uses getters to map `id → key`,` name → label`, `locked → disabled`.
  - Ideal when working with API-driven data.

<DynamicItemsMenu />

```tsx
const DynamicItemsMenu = () => {
  const dynamicItems = [
    { id: 1, name: "First", locked: false },
    { id: 2, name: "Second", locked: true }
  ];

  return (
    <Menu
      items={dynamicItems}
      getItemKey={(item) => String(item.id)}
      getItemLabel={(item) => item.name}
      getItemDisabled={(item) => item.locked}
    />
  );
};
```

#### Size variants

Menu supports different sizes for all items, allowing you to adjust spacing and font size consistently across the component.
  - `size` sets the uniform size for all menu items.
  - Available values: `"l" | "m" | "s" | "xs"`.

#### Best practices

1. Choose direction based on context
  - Use `horizontal` for top navigation bars.
  - Use `vertical for sidebars and dropdowns.
2. Indicate current location
  - Always mark the current page with `active`.
  - Improves orientation and user flow.

3. Icons and badges
  - Place icons on the left for visual hierarchy.
  - Use `rightContent` for badges or counters.

4. Disabled states
  - Use `disabled` to indicate unavailable actions.
  - Avoid removing items completely—disabled states help communicate intent.

5. Dynamic data
  - Use getter props (`getItemLabel`, `getItemDisabled`, etc.) to integrate with any backend response shape.
