- The Home item is highlighted as active.

#### Dynamic items with custom getters

Adapts the Menu to a custom data model.

- Uses getters to map `id ‚Üí key`,` name ‚Üí label`, `locked ‚Üí disabled`.
- Ideal when working with API-driven data.

<DynamicItemsMenu />

```tsx
const DynamicItemsMenu = () => {
  const dynamicItems = [
    { id: 1, name: "First", locked: false },
    { id: 2, name: "Second", locked: true }
  ];

  return (
    <Menu
      items={dynamicItems}
      getItemKey={(item) => String(item.id)}
      getItemLabel={(item) => item.name}
      getItemDisabled={(item) => item.locked}
    />
  );
};
```

#### Best practices

1. Choose the right orientation

- Use `vertical` for navigation sidebars, lists, and hierarchical menus.
- Use `horizontal` for top navigation or toolbars.

2. Keep labels concise

- Use short, clear text for label (e.g., ‚ÄúSettings‚Äù instead of ‚ÄúUser Settings and Preferences‚Äù).
- Avoid wrapping or overflowing text unless necessary.

3. Use icons for clarity, not decoration

- Add leftContent for commonly recognized symbols (‚öôÔ∏è for settings, üîí for security).
- Place rightContent for secondary actions or indicators (e.g., arrows, badges).

4. Manage states consistently

- Apply `activeKeys` to highlight the current selection.
- Use `disabled` for unavailable actions, keeping disabled items visible but not interactive.

5. Dynamic data

- Use getter props (`getItemLabel`, `getItemDisabled`, etc.) to integrate with any backend response shape.

6. Leverage nested items for structure

- Use `children` to group related actions under a parent menu item.
- Keep nesting shallow (1‚Äì2 levels) to maintain readability and usability.
