#### Basic usage

```tsx
<Modal
  isOpen={modalOpen}
  onClickClose={() => setModalOpen(false)}
  title="Confirmation"
  description="Are you sure you want to proceed?"
  footer={<Button onClick={() => setModalOpen(false)}>Yes</Button>}
/>
```

#### Best practices

- Always include title — gives users context and improves accessibility.
- Use description sparingly — only when additional clarity is needed.
- Use footer for primary actions — like "Save", "Cancel", or "Confirm".
- isCloseButton control — hide the close icon only in passive modals, otherwise provide clear exit controls.
- Enable onEsc — Escape key is a standard close action; ensure handler exists for consistency.
- Leverage isFullScreen wisely — use fullscreen modals for mobile or immersive tasks; avoid for simple prompts.
- Consistent sizing — stick to design-system sizes (sm/md/lg) unless use-case needs custom styling.
- Custom className only for advanced layout — most styling should come from tokens and theme, not CSS hacks.
- Handle stacking with zIndex — use only when stacking multiple overlays, otherwise let default work.
