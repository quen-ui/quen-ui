#### Basic usage

To enable notifications in your app:

  - Render `NotificationInstance` inside your application root.
  - Wrap with `QuenUIProvider` for proper theming.
  - Only one `NotificationInstance` should exist in the DOM.

```tsx
import { NotificationInstance } from "@quen-ui/components";
import { QuenUIProvider } from "@quen-ui/theme";

function Demo() {
  return (
    <QuenUIProvider>
      <NotificationInstance />
      {/* Your app here */}
    </QuenUIProvider>
  );
}
```

Once set up, you can trigger notifications anywhere in your app:

```tsx
export const NotificationsDefault = () => {
  const onClick = () => {
    notifications.show({
      title: "Default Notification",
      message: "Do not forget to star QuenUI on GitHub!"
    });
  };
  return <Button onClick={onClick}>Show notification</Button>;
};
```

#### Features

The `notifications` API provides the following functions:

  - `show(params)` - Adds a new notification
  - `hide(id)` - Removes a specific notification by ID
  - `update(params)` - Updates an existing notification
  - `clean` - Clears all notifications from the screen

```ts
  /** Displays new notification */
  show: (notification: INotificationParams) => string;
  /** Removes specific notification */
  hide: (id: string) => string;
  /** Updates existing notification */
  update: (notification: Partial<INotificationParams>) => string;
  /** Clears all notifications */
  clean: () => void;
```

These functions allow you to manage notifications dynamically throughout the app.

```tsx
const NotificationsPositions = () => {
  const positions = [
    "top-left",
    "top-right",
    "top",
    "bottom-left",
    "bottom-right",
    "bottom"
  ] as const;

  const onClick = (position: (typeof positions)[number]) => {
    notifications.show({
      title: `Notification at ${position}`,
      message: "Notification message",
      position
    });
  };
  return (
    <Flex gap="m">
      {positions.map((position) => (
        <Button key={position} onClick={() => onClick(position)}>
          Show {position} notification
        </Button>
      ))}
    </Flex>
  );
};
```

#### Best practice

- Status selection guide

  - `success`: For completed actions (e.g., payments, submissions)
  - `error`: For failures or blocking issues
  - `warning`: For non-critical problems (e.g., deprecations)
  - `info`: For neutral system messages

- Timing

  - Success / Info: `autoClose=5000`
  - Warnings: `autoClose=10000`
  - Errors requiring user action: `autoClose=false`
