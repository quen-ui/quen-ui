#### Basic pagination

Displays default pagination with page numbers and navigation controls.

<Pagination
  total={100}
  pageSize={10}
  onChange={(page) => console.log("Page:", page)}
/>

```tsx
<Pagination
  total={100}
  pageSize={10}
  onChange={(page) => console.log("Page:", page)}
/>
```

#### Pagination without controls

Renders only page numbers without "previous" and "next" controls.

<Pagination total={50} pageSize={5} withControls={false} />

```tsx
<Pagination total={50} pageSize={5} withControls={false} />
```

#### Pagination without pages

Shows only previous/next controls without individual page buttons.

<Pagination total={30} pageSize={10} withPages={false} />

```tsx
<Pagination total={30} pageSize={10} withPages={false} />
```

#### Custom rendering (`itemRender`)

Completely customizes how pagination items are rendered.

<Pagination
  total={100}
  pageSize={10}
  itemRender={(page, type) => {
    if (type === "prev") return <button>Previous</button>;
    if (type === "next") return <button>Next</button>;
    if (type === "dots") return <span>...</span>;
    return <button>{page}</button>;
  }}
/>

```tsx
<Pagination
  total={100}
  pageSize={10}
  itemRender={(page, type) => {
    if (type === "prev") return <button>Previous</button>;
    if (type === "next") return <button>Next</button>;
    if (type === "dots") return <span>...</span>;
    return <button>{page}</button>;
  }}
/>
```

#### Best Practices

  - Use `withControls={false}` on mobile – prevents UI clutter when screen space is limited.
  - Leverage `onChange` for API calls – fetch data when the page changes instead of loading all at once.
  - Customize with `itemRender` – to match brand guidelines or integrate with custom components.
  - Don’t render for empty data – if `total <= 0`, the component automatically returns `null`.
  - Combine with Table or List – pagination is often used alongside data display components.
