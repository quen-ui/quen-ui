#### Basic Usage

A standard skeleton with a basic width and height.

<Skeleton width={120} height={160} />

```tsx
import { Skeleton } from "@quen-ui/components";

export default () => <Skeleton width={120} height={160} />;
```

#### Skeleton with animation

There are two animation modes available: `pulse`, `shimmer`.

<Flex direction="column" gap="m">
  <Skeleton width={200} height={20} animation="pulse" />
  <Skeleton width={200} height={20} animation="shimmer" />
  <Skeleton width={200} height={20} />
</Flex>


#### Using loading to wrap content

When `loading = true`, the skeleton is displayed - otherwise the content.

<SkeletonLoading />

```tsx
const SkeletonLoading = () => {
  const [loading, setLoading] = useState(true);

  return (
    <Flex direction="column" gap="l">
      <Skeleton loading={loading} width={250} height={20}>
        <Text>Loaded content</Text>
      </Skeleton>
      <Button onClick={() => setLoading((prev) => !prev)}>
        Toggle Skeleton
      </Button>
    </Flex>
  );
};
```

#### Skeleton.Card

Pre-configured placeholder card.

<SkeletonCard />

```tsx
const SkeletonCard = () => {
  const [loading, setLoading] = useState(true);
  return (
    <Flex direction="column" gap="l">
      <Skeleton.Card
        loading={loading}
        width={262}
        height={180}
        animation="shimmer">
        <Card
          cover={
            <img
              alt="example"
              src="https://gw.alipayobjects.com/zos/rmsportal/JiqGstEfoWAOHiTxclqi.png"
            />
          }>
          <Text>Card</Text>
        </Card>
      </Skeleton.Card>
      <Button onClick={() => setLoading((prev) => !prev)}>
        Toggle Skeleton
      </Button>
    </Flex>
  );
};
```

##### Skeleton.Layout

A powerful tool for creating complex skeleton UI structures.
Allows you to pass a schema that describes the structure of a card or interface element.

<Skeleton.Layout
  loading
  direction="column"
  schema={{
    avatar: { size: "m" },
    title: 1,
    text: 3,
    button: { width: 160, size: "m" }
  }}
/>

```tsx
<Skeleton.Layout
  loading
  direction="column"
  schema={{
    avatar: { size: "m" },
    title: 1,
    text: 3,
    button: { width: 160, size: "m" }
  }}
/>
```

#### Best practices

  - Use `Skeleton.Layout` for most UI: it's flexible, repeatable, and supports layouts.
  - Skeletons should replicate the actual structure of the content as closely as possible.
  - Use `shimmer` for large visual UI loads, `pulse` for compact elements.
