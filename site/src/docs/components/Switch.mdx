import { Flex, Switch } from "@quen-ui/components";

#### Basic usage

The `Switch` is used to toggle a single setting on or off.
It supports both controlled (via `value`) and uncontrolled (via `defaultChecked`) usage.

<Switch defaultChecked label="Email notifications" size="l" />

```tsx
function Demo() {
  const [notifications, setNotifications] = useState(true);
  return (
    <Switch
      value={notifications}
      onChange={(checked) => setNotifications(checked)}
      label="Email notifications"
      size="l"
      defaultChecked
    />
  );
}
```

#### Label positioning

You can display the label before or after the switch. By default, the label is rendered after the toggle.

<Switch label="Dark Mode" />
<br />
<Switch label="Dark Mode" labelPosition="after" />

```tsx
// Label before switch
<Switch label="Dark Mode" />

// Label after switch (default)
<Switch label="Dark Mode" labelPosition="after" />
```

#### Disabled state

Use `isDisabled` to make the switch non-interactive. This is useful in read-only forms or when an option is not currently available.

<Switch label="Advanced mode" isDisabled size="s" />

```tsx
<Switch label="Advanced mode" isDisabled size="s" />
```

#### Size variants

Switch supports multiple sizes (`xs`, `s`, `m`, `l`). Use larger sizes for touch devices or when accessibility is a concern.

  <Switch label="Size S" size="s" />
  <Switch label="Size XS" size="xs" />
</Flex>

#### Best practices

##### When to use

  - For binary on/off settings
  - For immediate actions (e.g. toggling dark mode, notifications)
  - Inside forms when a boolean value is needed

##### When not to use

  - For multi-choice selections — use radio buttons or checkboxes instead
  - For actions that require confirmation — use a button or dialog
  - Without context — always provide a label or surrounding description
