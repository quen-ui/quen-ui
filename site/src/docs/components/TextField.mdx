#### Basic Text field

A simple text input with a label and placeholder. This is the default configuration for most text inputs.

<TextField placeholder="Enter your name" label="Full Name" />

```tsx
<TextField placeholder="Enter your name" label="Full Name" />
```

#### With icons and clear button

Add adornments for context and enable `isClearable` to let users reset input quickly. This pattern is especially useful for search bars or filters.

<TextField leftContent={<IconSearch />} isClearable placeholder="Search..." />

```tsx
<TextField
  leftContent={<IconSearch />}
  isClearable
  placeholder="Search..."
  onChange={(value) => setQuery(value)}
  value={query}
/>
```

#### Validation state

Mark inputs as required with `isRequired` and provide inline error feedback using `error`. The error can be a boolean (invalid state only) or a string (message).

<TextField
  label="Email"
  isRequired
  error="Invalid email"
  placeholder="user@example.com"
/>

```tsx
<TextField
  label="Email"
  isRequired
  error={errors.email ? "Invalid email" : undefined}
  value={email}
  onChange={(val) => setEmail(val)}
  placeholder="user@example.com"
/>
```

#### Password field

For passwords, you can toggle visibility using a button in `rightContent`. This improves usability without compromising security.

<TextFieldPassword />

```tsx
const [showPassword, setShowPassword] = useState(false);

<TextField
  label="Password"
  type={showPassword ? "text" : "password"}
  rightContent={
    <Button
      icon={showPassword ? <EyeOffIcon /> : <EyeIcon />}
      onClick={() => setShowPassword(!showPassword)}
    />
  }
/>;
```

#### Disabled state

Use `isDisabled` when the field should be visible but not editable. This is common in read-only forms or when the value is prefilled and locked.

<TextField label="Locked field" value="Prefilled value" isDisabled />

```tsx
<TextField label="Locked field" value="Prefilled value" isDisabled />
```

#### Size variants

#### Controlled usage

For form handling, use `value` and `onChange` to make the field controlled. This ensures synchronization with your state management.

```tsx
const [name, setName] = useState("");

<TextField
  label="Controlled"
  value={name}
  onChange={(val) => setName(val)}
  placeholder="Enter text"
/>
```

#### Best practices

###### 1. Validation patterns

  - Use error as a boolean to only show invalid styling.
  - Use error as a string when a message should be displayed.

```tsx
// Simple error display
<TextField error />

// // Text error display
<TextField error={errorMessage} />
```

###### 2. Clearable Fields

  - Best for search inputs or fields with temporary content.
  - Combine `isClearable` with `onClear` to reset state programmatically.

###### 3. Adornment Usage

  - Use leftContent for contextual icons (search, country code).
  - Use rightContent for actions (password toggle, units, counters).
  - Keep adornments simple to avoid clutter.
