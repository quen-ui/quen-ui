#### Basic Textarea

A simple textarea with placeholder and label. Use this as the default configuration for most form inputs.

<Textarea placeholder="Enter description" label="Project details" />

```tsx
<Textarea placeholder="Enter description" label="Project details" />
```

##### Auto-resizing with limits

Enable `autosize` to make the field grow dynamically as the user types. Use `minRows` and `maxRows` to define vertical expansion boundaries.

<Textarea
  autosize
  minRows={3}
  maxRows={8}
  placeholder="Content grows as you type..."
/>

```tsx
<Textarea
  autosize
  minRows={3}
  maxRows={8}
  placeholder="Content grows as you type..."
/>
```

##### Validation state

Display validation errors by passing a boolean or string to the `error` prop. Use `required` to mark the field as mandatory.

<Textarea label="Bio" required error={"Required field"} />

```tsx
<Textarea label="Bio" required error={"Required field"} />
```

##### Adornments + clearable

You can add left or right content (icons, buttons) and enable a clear button. This is useful for filters, searches, or fields with quick reset functionality.

<Textarea
  leftContent={<IconLock />}
  isClearable
  placeholder="Search descriptions..."
/>

```tsx
<Textarea
  leftContent={<IconLock />}
  isClearable
  placeholder="Search descriptions..."
/>
```

#### Disabled state

Use `disabled` to make the textarea non-editable. This is useful when the content is read-only or locked.

<Textarea label="Locked notes" placeholder="Content not editable" disabled />

```tsx
<Textarea label="Locked notes" placeholder="Content not editable" disabled />
```

#### Controlled usage

Use `value` and `onChange` for controlled forms. This allows you to fully manage the state of the textarea.

```tsx
const [bio, setBio] = useState("");

<Textarea label="Controlled bio" value={bio} onChange={(val) => setBio(val)} />;
```

#### With right adornment

You can also add `rightContent` for actions such as counters or icons.

<Textarea placeholder="Type something..." rightContent={<span>0/200</span>} />

```tsx
<Textarea placeholder="Type something..." rightContent={<span>0/200</span>} />
```

#### Character counter

Combine `maxLength` with `rightContent` to show a live counter. This is useful for bios, messages, or any input with a limit.

```tsx
const [text, setText] = useState("");
<Textarea
  label="Bio"
  value={text}
  onChange={(val) => setText(val)}
  maxLength={200}
  rightContent={<span>{text.length}/200</span>}
/>;
```

#### Best practices

###### 1. Auto-resizing configuration

Use small ranges (`minRows={2}`–`maxRows={6}`) for short inputs like comments. - Use larger ranges (`minRows={4}`–`maxRows={12}`) for long inputs like documentation.

```tsx
// Good for comments
<Textarea autosize minRows={2} maxRows={6} />

// Good for long content
<Textarea autosize minRows={4} maxRows={12} />
```

###### 2. Validation patterns

Use `error` as a boolean for generic invalid state. - Use `error` as a string to display specific error messages.

```tsx
// Simple error display
<Textarea error />

// // Text error display
<Textarea error={errorMessage} />
```

###### 3. Content Guidelines

- Use `minRows` to hint expected length of input.
- Set `maxRows` to prevent excessive vertical scrolling.
- Combine with `isClearable` for user-generated content (search, notes).
- Add counters with `rightContent` for length-restricted inputs.
