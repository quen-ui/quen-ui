#### Basic usage

Renders a heading with default appearance. The `size` prop is required and defines the visual hierarchy.

<Title size="l">Section Title</Title>

```tsx
<Title size="l">Section Title</Title>
```

#### Size variants

Control heading hierarchy using predefined sizes: `2xl`, `xl`, `l`, `m`, `s`, `xs`.

#### Custom color

The `color` prop overrides the default or type-based style.

  Custom colored heading
</Title>


#### Semantic element

Use the `as` prop to render different HTML tags (`h1`–`h6`, `div`, `span`, etc.), while keeping consistent styles.

  </Title>
  <Title as="h2" size="xl">
    Section Title (h2)
  </Title>
</Flex>

#### Clickable heading

Headings can also be interactive by passing an `onClick` handler.

<Title size="m" onClick={() => alert("Heading clicked!")}>
  Clickable Title
</Title>

```tsx

  Clickable Title
</Title>
```

#### Best practices

1. Hierarchy & sizes
  - Use `2xl` and `xl` for page and main section titles.
  - Use `l` and `m` for subsections.
  - Use `s` and `xs` for minor headings, captions, or card titles.

2. Semantic structure
  - Match `as` with document structure:
    - Page title → h1
    - Section title → h2
    - Subsection → h3 or smaller
  - Keep hierarchy logical for accessibility (don’t skip levels).

3. Status usage
  - `success` → success states (e.g., confirmation title).
  - `warning` → cautionary notices.
  - `danger` → destructive or error-related titles.
  - `disabled` → muted headings in disabled sections.

4. Interactivity
  - Use `onClick` only when the title performs an action (like expanding a panel).
  - Avoid making primary headings clickable unless they serve as navigation.
