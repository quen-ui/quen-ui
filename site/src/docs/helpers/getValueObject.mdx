#### Signature

```ts
function getValueObject<
  D,
  P extends TKeyObjectType<D>,
  DV = TValueObjectType<D, P>
>(
  data: D,
  path: P,
  defaultValue?: DV
): TValueObjectType<D, P> | DV

```

#### Return Value

- Returns the value at the specified path if it exists.
- If the path resolves to `undefined`, returns the provided `defaultValue`.

#### Usage Examples

##### Accessing a simple nested property

```ts
import { getValueObject } from '@quen-ui/helpers';

const data = { user: { name: "Alice", age: 25 } };

const name = getValueObject(data, "user.name");
// "Alice"
```

Explanation:
  - The function follows the path "user.name" and returns "Alice".

##### Accessing an array element

```ts
import { getValueObject } from "@quen-ui/helpers";

const data = { users: [{ id: 1, name: "Alice" }, { id: 2, name: "Bob" }] };

const firstUserName = getValueObject(data, "users[0].name");
// "Alice"
```

Explanation:
  - The path `users[0].name` correctly resolves to the first user's name.e.

##### Missing path returns undefined

```ts
import { getValueObject } from "@quen-ui/helpers";

const data = { user: { name: "Alice" } };

const age = getValueObject(data, "user.age");
// undefined
```

Explanation:
 - `user.age` does not exist in `data`, so the result is `undefined`.

##### Providing a default value

```ts
import { getValueObject } from "@quen-ui/helpers";

const data = { user: { name: "Alice" } };

const age = getValueObject(data, "user.age", 30);
// 30
```

Explanation:
  - Since `user.age` is missing, the function returns the provided default `30`.

##### Handling deeply nested arrays

```ts
import { getValueObject } from "@quen-ui/helpers";

const data = {
  groups: [
    { members: [{ name: "Alice" }, { name: "Bob" }] },
    { members: [{ name: "Charlie" }] }
  ]
};

const memberName = getValueObject(data, "groups[0].members[1].name");
// "Bob"
```

Explanation:
  - The function traverses arrays and objects seamlessly with mixed notation.
