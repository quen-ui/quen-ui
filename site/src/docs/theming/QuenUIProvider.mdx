The QuenUIProvider is the foundation for customizing and theming your QuenUI application. This context provider delivers your theme configuration to all QuenUI components, ensuring visual consistency and centralized styling across your application.

#### Features

- **Theme Propagation:** Distributes theme settings to all child components
- **Customization Hub:** Centralized control over colors, spacing, typography, and components
- **Type Safety:** Full TypeScript support for theme structure
- **Styled Components Integration:** Built on styled-components theming API
- **Component Overrides:** Customize individual component styles
- **Per-Component Tokens:** Fine-grained control over visual tokens for each QuenUI component
- **Common Tokens:** Global shared tokens for consistent color semantics (success, danger, etc.)

#### Installation

```bash
npm i @quen-ui/theme
```

#### Basic usage

```tsx
import { QuenUIProvider, createTheme } from "@quen-ui/theme";
import { Button } from "@quen-ui/components";

function App() {
  const customTheme = createTheme({
    /** Your theme override here */
  });
  return (
    <QuenUIProvider theme={customTheme}>
      <Button>Styled Button</Button>
    </QuenUIProvider>
  );
}
```

#### Theme structure (IQuenUITheme)

The theme object follows this structure:

```ts
export interface IQuenUITheme<C extends TQuenUIColors = TQuenUIColors> {
  /** Color palette */
  colors: C;

  /** Primary brand color */
  primaryColor: keyof TQuenUIColors;

  /** Body background color */
  colorBody: string;

  /** Default text color */
  textColor: string;

  /** Base font family */
  fontFamily: string;

  /** Control styles (radius, height) */
  control: IQuenUIThemeControl;

  /** Spacing system */
  space: IQuenUIThemeSpace;

  /** Typography system */
  fonts: IQuenUIFont;

  /** Component-specific tokens */
  components: IQuenUIComponents | Record<string, any>;

  /** Common color tokens shared across components */
  commonColorTokens: IQuenUICommonTokensColor;
}
```

#### Component Tokens (`theme.components`)

The `components` section allows you to override or extend visual tokens for each QuenUI component individually.
Each component (e.g. `Button`, `Card`, `Input`, etc.) has its own set of tokens generated via the `generateComponentTokens(theme)` function.
You can customize these tokens globally in your theme:

```ts
import { createTheme } from "@quen-ui/theme";

const customTheme = createTheme({
  components: {
    Button: {
      successBackground: "#4B6BFB",
      color: "#fff"
    },
    Card: {
      radius: "12px",
      borderColor: "rgba(0, 0, 0, 0.1)"
    }
  }
});
```

These tokens are automatically consumed by all QuenUI components, providing consistent yet flexible styling control.

#### Best practices

- Consistent Naming: Use the same token names across components
- Spacing Scale: Use multiples of 4px or 8px for spacing

#### Advanced techniques

##### Theme variants

```ts
import { createTheme, QuenUIDarkTheme } from "@quen-ui/theme";

type ThemeVariant = "default" | "compact" | "luxury";

function createNewTheme(variant: ThemeVariant): IQuenUITheme {
  switch (variant) {
    case "compact":
      return createTheme({ space: compactSpacing });
    case "luxury":
      return createTheme({ colors: premiumColors }, QuenUIDarkTheme);
    default:
      return baseTheme;
  }
}
```

##### Runtime theme updates

```tsx
function ThemeSwitcher() {
  const [theme, setTheme] = useState(QuenUILightTheme);

  const toggleTheme = () => {
    setTheme((current) =>
      current === QuenUILightTheme ? QuenUIDarkTheme : QuenUILightTheme
    );
  };

  return (
    <QuenUIProvider theme={theme}>
      <Button onClick={toggleTheme}>Toggle Theme</Button>
      <AppContent />
    </QuenUIProvider>
  );
}
```

#### useTheme

Hook to access theme object in components:

```tsx
import { useTheme } from "@quen-ui/theme";

function ThemedComponent() {
  const theme = useTheme();
  return <div style={{ color: theme.colors[theme.primaryColor][5] }} />;
}
```
